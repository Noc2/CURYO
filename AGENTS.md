# Repository Instructions

## Commit Policy
- After completing any code or documentation change, create a git commit automatically unless the user explicitly says not to.
- Commit only the files relevant to the current task.
- Do not include unrelated dirty worktree changes in the commit.
- Use a concise, descriptive commit message.
- If pre-commit checks fail, fix the issue when reasonable and then commit.
- If a commit would be unsafe because of ambiguous or conflicting local changes, stop and explain why instead of forcing it.
